You are a Senior Software Architect preparing a Data Flow Diagram (DFD) decomposition.

You have been given software requirements extracted from Glassdoor user reviews for a workplace review web application.

REQUIREMENTS:
{requirements_json}

YOUR TASK:
Identify all four DFD Level-1 components:

1. EXTERNAL ENTITIES — people or systems OUTSIDE the application that send data TO or receive data FROM it.
2. PROCESSES          — internal functions the system performs (numbered, e.g. P1, P2 …).
3. DATA FLOWS         — named arrows between entities, processes, and data stores (format: Source → Destination: DataName).
4. DATA STORES        — persistent storage locations (named D1, D2 … e.g. D1: User Profiles).

RULES:
- Every External Entity must have at least one data flow in and one data flow out.
- Every Process must have at least one input flow and one output flow.
- Every Data Store must be read by at least one process and written by at least one process.
- Derive components ONLY from what the requirements describe — do not invent functionality.
- Use clear, short names (2–5 words) for each component.
